## 🎯 ระบบนี้คืออะไร?

ระบบนี้เป็นระบบตอบคำถามอัจฉริยะที่ใช้เทคโนโลยี AI ขั้นสูง สามารถค้นหาข้อมูลจากเอกสารและตอบคำถามได้อย่างแม่นยำ โดยเฉพาะเหมาะสำหรับ:

- 📚 ระบบคำถาม-คำตอบจากเอกสารทางการแพทย์
- 🔍 การค้นหาข้อมูลที่ซับซ้อนข้ามหลายเอกสาร
- 💡 การวิเคราะห์และเปรียบเทียบข้อมูล
- 🇹🇭 รองรับภาษาไทยเต็มรูปแบบ

## ✨ จุดเด่น

### 1. ค้นหาแม่นยำด้วย Hybrid Search
- ผสมผสานการค้นหาแบบคำสำคัญ (BM25) และความหมาย (Vector Search)
- ได้ผลลัพธ์ที่ตรงประเด็นมากขึ้น

### 2. เข้าใจบริบทด้วย Contextual Retrieval
- เพิ่มบริบทให้กับข้อมูลแต่ละส่วน
- ลดการตอบผิดหรือตอบไม่ตรงคำถาม

### 3. ตอบคำถามได้หลากหลาย
- คำถามเปรียบเทียบ (เช่น เปรียบเทียบโรค A กับ B)
- คำถามเชิงวิเคราะห์ (เช่น ทำไมถึงเกิดอาการนี้)
- คำถามเชิงสถิติ (เช่น มีผู้ป่วยกี่คน)
- คำถามเฉพาะเจาะจง (เช่น ขนาดยาเท่าไหร่)

### สิ่งสำคัญที่ต้องรู้

**Volume Mapping:**
- ไฟล์ใน `corpus_input/` ถูก **share** ระหว่าง host และ containers
- **Streamlit UI** มีสิทธิ์ **อ่าน-เขียน** (สามารถเพิ่ม/แก้ไข/ลบได้)
- **MCP Server** มีสิทธิ์ **อ่านอย่างเดียว** (ใช้สำหรับ indexing)
- การเปลี่ยนแปลงจะ **sync อัตโนมัติ** ระหว่าง host และ containers

**หลังจากเพิ่มไฟล์:**
1. ไฟล์จะปรากฏใน Web UI ทันที
2. ต้อง **index เอกสารใหม่** เพื่อให้ค้นหาได้
3. ใช้ MCP tool: `add_documents(path="/app/rag/corpus_input")`

**ดูรายละเอียดเพิ่มเติม:** [VOLUME_MAPPING_GUIDE.md](VOLUME_MAPPING_GUIDE.md)

### ตัวอย่างที่ 2: คำถามเชิงสถิติ

**คำถาม:** "ในปี 2010 มีผู้เสียชีวิตจากอหิวาตกโรคทั่วโลกเท่าไหร่"

**คำตอบ:**
```
ในปี 2010 มีผู้เสียชีวิตจากอหิวาตกโรคทั่วโลกประมาณ 100,000-130,000 คนต่อปี
```

### ตัวอย่างที่ 3: คำถามเฉพาะเจาะจง

**คำถาม:** "ยา Tetracycline ใช้รักษาอหิวาตกโรคในขนาดเท่าไหร่"

**คำตอบ:**

```
ขนาด: 2 กรัม เพียงครั้งเดียว 
หรือ 500 มิลลิกรัม วันละ 4 ครั้ง ติดต่อกัน 3 วัน

สำหรับเด็กและหญิงตั้งครรภ์ ใช้ยาทางเลือก:
```

## ❓ คำถามที่พบบ่อย (FAQ)

### Q: ควรใช้ Web UI หรือ Command Line?
A: 
- **Web UI**: แนะนำสำหรับผู้ใช้ทั่วไป ใช้งานง่าย มี interface สวยงาม
- **Command Line**: สำหรับผู้ที่ต้องการควบคุมเอง หรือใช้ใน script อัตโนมัติ

### Q: Web UI ต้องการอะไรบ้าง?
A: 
- Python 3.8+
- OpenSearch running (localhost:9200)
- Dependencies จาก requirements-ui.txt
- เบราว์เซอร์ (Chrome, Firefox, Safari)

### Q: ระบบรองรับภาษาอะไรบ้าง?
A: รองรับภาษาไทยเป็นหลัก และภาษาอังกฤษ

### Q: ต้องมี GPU ไหม?
A: ไม่จำเป็น แต่ถ้ามี GPU จะทำงานเร็วขึ้น

### Q: เอกสารต้องมีขนาดเท่าไหร่?
A: แนะนำไฟล์ละไม่เกิน 10 MB

### Q: ใช้เวลานานแค่ไหน?
A: 
- การสร้างฐานข้อมูล: 5-10 นาที (ขึ้นกับจำนวนเอกสาร)
- การค้นหาและตอบคำถาม: 2-3 วินาทีต่อคำถาม

### Q: ระบบตอบผิดทำอย่างไร?
A: 
1. ตรวจสอบว่าข้อมูลในเอกสารครบถ้วน
2. ลองปรับคำถามให้ชัดเจนขึ้น
3. เพิ่มเอกสารที่เกี่ยวข้อง

### Q: สามารถใช้กับเอกสารประเภทอื่นได้ไหม?
A: ได้ แต่ต้องแปลงเป็น Markdown ก่อน

## 🎉 เริ่มต้นใช้งานเลย!

### สำหรับผู้ใช้ทั่วไป (Web UI)

```bash
# 1. ติดตั้ง
cd AuthenticRAG-Qwen2.5API
pip install -r requirements-ui.txt

# 2. ทดสอบ
python test_ui_setup.py

# 3. เริ่มใช้งาน
./start_ui.sh

# 4. เปิดเบราว์เซอร์
# http://localhost:8501
```

### สำหรับผู้ใช้ขั้นสูง (Command Line)

```bash
# 1. ติดตั้ง
cd AuthenticRAG-Qwen2.5API && ./setup.sh

# 2. ทดสอบ
python demo_check_system.py

# 3. ใช้งาน
python onlysearchAuthenticRAG.py
```
